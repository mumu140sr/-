# 📅 シフト自動生成アプリ

焼きなまし法（Simulated Annealing）による高度な最適化エンジンを搭載したシフト作成Webアプリです。
ブラウザだけで動作し、Excel不要・サーバー不要で使えます。

## ✨ 主な機能

### 🎯 最適化エンジン
- 焼きなまし法による高度な組合せ最適化
- 複数の近傍生成戦略（個人内入替 / 個人間入替 / シフト変更 / 大規模入替）
- 局所最適脱出のためのリヒート機構
- 最大20万回の反復で最適解を探索
- **Web Worker でバックグラウンド実行** — UIをブロックせず、最適化中も操作可能
- **⏹中止ボタン** — 走っている最適化をいつでも停止可能
- Worker非対応環境では自動でメインスレッドにフォールバック

### 📋 設定可能なルール
- **連勤上限**（可変、デフォルト4日）
- **遅→早の禁止**（インターバル不足）
- **遅→研の禁止**（研修も早番時間帯のため）
- **連勤内での時間帯切替禁止**（早系で始まったら早系で揃える）
- **単発休みのペナルティ**（リズムの悪い休み、特に遅→休→早）
- **個人ごとの公休数の上限**
- **月またぎ**（前月末の連勤数＋最終シフト早/遅 を考慮）
- **スタッフごとの早番/遅番希望**
- **スタッフごとの早遅バランス比率**（早多め〜均等〜遅多め）
- **役職別の必要人数**（早責・遅責・早総務・遅総務・早・遅）
- **特別日**（入れ替え日＝副店長を遅責固定 / 新装日＝副店長を早責固定）

### 👥 役職タイプ × 役割タイプの2軸モデル
**役職タイプ**（組織上の地位）
- 副店長 / チーフ / リーダー / スタッフ

**役割タイプ**（業務上の役割）
- 責任者（早責・遅責）
- 一般（早・遅）
- 一般（営業）（早・遅）
- 総務（早総務・遅総務）

### 🔁 代替役割の優先順位制（不足時の自動代替）
**責任者の代替**: 副店長・チーフのみ（リーダー以下は不可）

**総務の代替**: 以下の優先度順で割当（ペナルティ軽 → 重）
1. 第一代替: リーダー職位 + 一般役割 (+20)
2. 第二代替: スタッフ職位 + 一般役割 (+50)
3. 第三代替: リーダー職位 + 一般営業役割 (+100)
4. 最終手段: チーフ (+300)

### 🎨 視覚的な操作
- 色分けされたシフト表（役職ごとに色が違う、カスタマイズ可能）
- ルール違反箇所の赤枠ハイライト
- **ドラッグ&ドロップで手動修正**（セル間でシフト交換）
- **セルクリックで直接編集**（モーダルでシフト種別を選択）
- カレンダー風の希望休入力（休系シフト + 出勤シフト指定の両対応）
- 手動編集時にリアルタイムで違反・カウント更新

### 📥 入出力
- Excelエクスポート（色付き、集計シート付き、研修列含む）
- CSVエクスポート（UTF-8 BOM付きでExcel対応）
- ブラウザのlocalStorageで自動保存（編集の都度デバウンス保存）
- 「保存」「読込」「リセット」ボタンで明示的操作も可

### 📊 AI診断レポート + 🤖 AI解説モーダル
- 違反種別ごとに集計（致命的/警告/情報）
- 違反箇所と推奨アクションを提示
- スタッフ別の勤務/休日数集計
- **「🤖 AIに違反を解説してもらう」ボタン**：違反をタイプ別にグルーピングし、
  原因分析と解決のヒントを自然言語で提示（完全ローカル処理・API不要）

## 🚀 使い方

1. `index.html` をブラウザで開く
2. **① 基本設定**で対象月・連勤上限などを決める
3. **② 役職マスター**で各シフトの必要人数を設定
4. **③ スタッフ管理**で各スタッフを登録（最大20人）
   - 役職タイプ・役割タイプ・公休数・早遅希望・早遅バランス
   - 前月末の連勤数・最終シフト（早/遅）も入力
5. **④ 希望休入力**で各スタッフの希望休を入力
   - 休系シフト（休/公/有/☆/季/引/慶）
   - 出勤シフト指定（早/遅/早責/遅責/早総務/遅総務/研）も同じUIから固定可能
   - カレンダーで特別日（入れ替え日/新装日）も設定
6. **⑤ 自動生成**ボタンを押す
   - 実行中も他のタブを操作できる
   - 進捗バー・スコア・経過秒数が表示される
7. **⑥ シフト表**で結果を確認、必要に応じてドラッグ&ドロップまたはクリックで修正
   - 編集の都度、必要人数・公休数が自動更新
8. **🤖 AI解説ボタン**で違反の原因と解決法をチェック
9. Excel / CSVエクスポートで保存

## 🌐 起動方法

### 方法A: ローカルファイルとして直接開く
`index.html` をダブルクリック → ブラウザで開く
※ Web Workerはブラウザ・OS によっては `file://` で動かないため、その場合は自動的にメインスレッド実行にフォールバックします。

### 方法B: 簡易HTTPサーバー経由（推奨）
```bash
cd /path/to/webapp
python3 -m http.server 8080
# → http://localhost:8080 にアクセス
```
Workerの恩恵を完全に受けるには、こちらの方法が推奨です。

## 📂 ファイル構成

```
.
├── index.html              # メインHTMLファイル
├── README.md               # このファイル
├── css/
│   └── style.css           # スタイルシート
└── js/
    ├── data.js             # データモデル・定数・localStorage管理
    ├── optimizer.js        # 焼きなまし法による最適化エンジン
    ├── optimizer.worker.js # Web Worker エントリポイント
    ├── worker-client.js    # メインスレッドからWorkerを呼ぶラッパー
    ├── ai-explain.js       # AI解説モーダル（違反の自然言語化）
    ├── ui.js               # UI描画・イベント処理
    └── app.js              # メインエントリ・初期化
```

### スクリプト読込順序（重要）
`index.html` 内では以下の順で読み込みます：
```html
<script src="js/data.js"></script>
<script src="js/optimizer.js"></script>
<script src="js/worker-client.js"></script>
<script src="js/ai-explain.js"></script>
<script src="js/ui.js"></script>
<script src="js/app.js"></script>
```
Worker は `js/optimizer.worker.js` を別途読み込み、その中で
`importScripts('data.js', 'optimizer.js')` しています。

## 📜 ルール詳細（最適化のスコア）

スコアは小さいほど良く、0が完璧。違反タイプ別のペナルティ：

| ルール | ペナルティ |
|--------|-----------|
| 必要人数不足 | × 10000 |
| 必要人数超過 | × 200 |
| 連勤上限超過 | 800 × 超過日数 + 200 × 超過日数² |
| 遅→早（研含む）禁止 | 1500 |
| 役割タイプ違反 | 50000 |
| 連勤内の時間帯切替 | 600 |
| 遅→休→早(研) | 200 |
| その他の単発休み | 50 |
| 単発出勤 | 80 |
| 公休数の目標差 | 300 × 差 |
| 早遅バランスのずれ | 80 × ずれ |
| 第一代替（リーダー+一般 → 総務） | 20 |
| 第二代替（スタッフ+一般 → 総務） | 50 |
| 第三代替（リーダー+一般営業 → 総務） | 100 |
| 最終手段（チーフ → 総務） | 300 |

## 💡 元になったVBAコード

このアプリは、Excel VBAで書かれた「シフト最適化エンジン」を元に、ブラウザ上で動作するように再実装したものです。VBAでは100人分のシートを処理していましたが、本アプリは最大20人を想定して、より高機能・高速にしています。
